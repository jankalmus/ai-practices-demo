# Feature Documentation — Index

This directory holds **business-feature documentation**: for each feature, what it must do, what it deliberately does not do, what it integrates with, and which files implement it. These are distinct from the engineering standards in [../index.md](../index.md) — standards say *how we build*; feature docs say *what we built and why*.

Each doc is authored and kept current by the `document` skill, rendered from a shared template so they read consistently. A feature doc's **Files** section is the inverse of the `@feature` headers carried by AI-authored `.ts`/`.tsx` files (see [../documentation-standards.md](../documentation-standards.md)): a file tagged `@feature <slug>` belongs to the doc for `<slug>`.
